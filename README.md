# 배포일정 캘린더

배포(RFC) 일정을 캘린더 화면에서 한눈에 확인하는 설치형(exe) 데스크톱 프로그램입니다.
공유 폴더에 있는 엑셀을 자동으로 읽어 캘린더에 표시하고, 엑셀이 갱신되면 자동으로
새로고침합니다.

- **기술 스택**: Electron + FullCalendar
- **데이터**: 공유 폴더 자동 감지 (`chokidar`) + 엑셀 파싱 (`xlsx` / SheetJS)
- **대상 OS**: Windows (exe 설치형)
- **팀**: 좌측 상단 토글로 **GMK / GMAP** 전환 (팀마다 별도 폴더)

---

## 빠른 시작

```bash
npm install         # 의존성 설치
npm start           # 개발 모드로 앱 실행
```

1. 좌측 상단 **GMK / GMAP** 토글에서 팀 선택
2. 우측 상단 **공유 폴더 설정**으로 그 팀의 엑셀이 있는 폴더 지정
3. 다른 팀도 토글 후 같은 방식으로 폴더 지정 — 끝
   (이후 활성 팀 폴더를 자동 감시 → 엑셀이 바뀌면 자동 갱신)

설정은 사용자별로 저장됩니다: `%APPDATA%\deploy-calendar\config.json`
(`{ active, teams: { GMK: {watchDir}, GMAP: {watchDir} } }`).
구버전 `{ watchDir }` 설정은 자동으로 GMK로 이전됩니다.

## 엑셀 형식

폴더 안의 모든 `.xlsx` / `.xls` / `.csv` 파일을 읽고, **시트 형식을 자동 감지**해서
아래 두 가지를 모두 지원합니다.

### (A) RFC 관리 시트 형식 — GMK / GMAP 실제 시트

월별 시트(`2026.06` 등)로 구성된 RFC 관리 시트를 인식합니다. GMK(기획팀)와
GMAP(캐딜락 M62A)이 동일한 형식입니다.

- **시트 선별**: 시트명이 `YYYY.MM` 인 것 중 **최근 6개월 + 미래**만 읽습니다
  (범위는 [src/main.js](src/main.js)의 `RECENT_MONTHS`로 조정).
- **배포일 = 섹션 헤더에서 추출**. 괄호 안 날짜가 LIVE 날짜보다 우선합니다.
  | 헤더 예시 | 캘린더 표기 |
  |-----------|-------------|
  | `<6/30 LIVE> ... (6/30(화) 09시~7/1(수) 19시)` | **6/30 ~ 7/1** 기간으로 연결 |
  | `<6/2 LIVE> ... (6/4(목) 09시~19시)` | 괄호 단일 날짜 → **6/4** |
  | `<4/3 LIVE> 4월 2일(목) RFC&UAT` (괄호 날짜 없음) | LIVE 날짜 → **4/3** |
- **업무(Task Code 또는 STATUS가 있는 행)마다 캘린더 바 1개**를 만듭니다.
  코드/상태가 둘 다 없는 소제목·연속행은 제외합니다.
- 같은 날 여러 건이면 그만큼 바가 쌓이고, 한 칸에 많으면 `+N more`로 접힙니다.
- **상태 컬럼 자동 감지**: 헤더가 `STATUS` / `Staus`(오타) 등 달라도 `요청중` 포함
  여부로 함께 인식합니다.
- 일정 클릭 시 상세 팝업에 **구분 / Task Code / 담당자 / 상태 / 메모 / 출처 시트**를 표시.
- **중복 제거**: 월별 시트가 월말/월초 경계에서 겹쳐 같은 배포가 인접 두 시트에
  들어 있는 경우, **Task Code 기준으로 한 건만** 표시합니다.

### (B) 단순 형식 — 새로 만들 때 권장

`날짜 / 제목 / 담당자 / 비고` 4컬럼이면 됩니다 (CSV도 동일).

| 날짜 | 제목 | 담당자 | 비고 |
|------|------|--------|------|
| 2026-06-16 | 주문 시스템 v2.3 배포 | 김철수 | 운영 반영, 무중단 |

- **날짜**: `2026-06-16`, `2026.06.16`, `2026/6/16`, 엑셀 날짜 셀 모두 인식
- 컬럼명은 유연하게 인식 (날짜=`배포일`, 제목=`시스템명` 등). [src/main.js](src/main.js) `COLUMN_ALIASES`에서 조정.
- **CSV 인코딩**: UTF-8과 CP949(한국 Excel 기본 저장) 모두 자동 인식. 별도 변환 불필요.

샘플 엑셀 생성:

```bash
npm run sample      # sample-data/배포일정_샘플.xlsx 생성
```

## 공유 폴더 연결 (OneDrive)

이 프로그램은 **로컬 폴더 경로만** 읽습니다(SharePoint 웹 직접 접근 X). 따라서
SharePoint/Teams 문서를 **OneDrive로 동기화해 로컬 폴더로 만든 뒤** 그 폴더를 지정합니다.

```
SharePoint(웹)  ──OneDrive 동기화──▶  로컬 폴더  ──▶  프로그램이 읽음
```

**팀원이 PC마다 한 번만** 설정하면 됩니다:

1. SharePoint/Teams에서 대상 폴더(예: `11. RFC`, `06. RFC`) 열기
2. 상단 **바로 가기 추가(Add shortcut to OneDrive)** 또는 **동기화(Sync)** 클릭 → 회사 계정 로그인
3. 탐색기에 생긴 로컬 폴더(예: `C:\Users\<이름>\OneDrive - 회사\... - 11. RFC`)를
   프로그램의 **공유 폴더 설정**에서 지정

한 번 연결하면 이후엔 자동입니다. 누가 SharePoint에서 엑셀을 수정하면 OneDrive가
각 PC로 내려받고, 프로그램이 변경을 감지해 갱신합니다. (동기화에 수 초~수 분 지연이
있을 수 있으며, 급하면 우측 상단 **새로고침** 버튼으로 즉시 다시 읽습니다.)

> 대안: 사내 **네트워크 드라이브/공유 폴더**에 엑셀을 두고 모두 그 경로를 지정하면
> OneDrive 없이도 됩니다.

## exe 설치파일 만들기

```bash
npm run dist        # release/ 폴더에 NSIS 설치파일(.exe) 생성
```

- 출력: `release\배포일정 캘린더 Setup 1.0.0.exe`
- 팀원에게 이 **Setup .exe** 하나만 주면 설치됩니다(설치 폴더 선택·바탕화면 바로가기).
- 설치한 앱은 설정 저장 위치가 개발용과 달라(`%APPDATA%\배포일정 캘린더`),
  **처음 실행 시 GMK/GMAP 공유 폴더를 각각 한 번씩 지정**하면 됩니다.

### ⚠️ Windows 빌드 권한 (중요)

electron-builder가 서명 도구(winCodeSign)를 풀 때 **심볼릭 링크 생성 권한**이 필요합니다.
일반 권한이면 `Cannot create symbolic link ... 클라이언트가 필요한 권한을 가지고 있지
않습니다` 오류로 빌드가 실패합니다. 아래 중 하나를 **한 번만** 해주세요.

- **개발자 모드 켜기(추천)**: 설정 → 개인정보 및 보안 → 개발자용 → **개발자 모드 켬**
- 또는 **관리자 권한 터미널**에서 `npm run dist` 실행

> 설치 없이 폴더 형태로만 빌드: `npm run pack` (→ `release\win-unpacked`). 이것도 위
> 권한이 필요합니다.

## 프로젝트 구조

```
src/
  main.js        메인 프로세스: 설정·팀 전환, 폴더 감시, 엑셀 파싱, IPC
  preload.js     contextBridge로 안전한 API 노출
  index.html     화면 레이아웃 (팀 토글 포함)
  renderer.js    FullCalendar 렌더링 + 상세 팝업
  styles.css     스타일
scripts/
  make-sample.js 샘플 엑셀 생성기
```

## 동작 방식

1. 앱 시작 → 활성 팀(GMK/GMAP) 폴더의 엑셀을 읽어 캘린더에 표시
2. `chokidar`가 활성 팀 폴더의 엑셀 추가/수정/삭제를 감지 → 자동 새로고침
   (쓰기가 끝나고 0.8초 안정된 뒤 읽어 저장 중 깨짐 방지)
3. 팀 토글 전환 시 해당 팀 폴더를 최신으로 다시 읽음
4. 일정 클릭 → 구분 / Task Code / 담당자 / 상태 / 메모 / 출처 시트 상세 팝업
